# Development Phases (phase.md)

이 문서는 AI 코딩 에이전트가 순차적으로 프로젝트를 빌드하기 위한 마이크로 태스크 지침서입니다. 프론트엔드는 Next.js 15 (App Router), 백엔드는 Google Cloud Run (FastAPI + GNU Go) 환경을 사용합니다. 각 하위 단계(예: 1-1, 1-2)가 완전히 작동하는지 테스트한 후 다음 단계로 넘어가세요.

## Phase 1: 로그인 스크린 및 UI 개편

- **목표:** 아이들이 친근감을 느낄 수 있는 직관적인 로그인 화면 구축.
- ✅ **1-1. 로고 배치:** `app/page.tsx` 화면 중앙에 `next/image`를 사용하여 `assets/images/bbulmung_logo.png`를 반응형으로 배치.
- ✅ **1-2. 타이틀 적용:** 로고 상단에 "뿔멍바둑"이라는 텍스트를 Jua 웹 폰트를 적용하여 크고 눈에 띄게 렌더링.
- ✅ **1-3. 입력 폼 컴포넌트:** 로고 하단에 비밀번호(코드) 입력 창과 '시작하기' 버튼 UI 구현.
- ✅ **1-4. 인증 로직 구현:** 입력된 비밀번호가 환경변수(`.env.local`의 `NEXT_PUBLIC_APP_PASSWORD`)와 일치하면 `localStorage`에 인증 상태를 저장하고 `/menu`로 라우팅.

## Phase 2: 게임 메뉴 및 조건부 접바둑 UI 구현

- **목표:** 난이도(레벨)와 접바둑 개수를 설정하는 메인 메뉴 구성.
- ✅ **2-1. 메뉴 레이아웃:** `app/menu/page.tsx` 생성 및 바둑판 크기(9x9, 13x13, 19x19) 선택 UI 구현.
- ✅ **2-2. 대전 레벨 선택기:** 레벨 1부터 10까지 선택할 수 있는 버튼 그룹 컴포넌트 구현.
- ✅ **2-3. 조건부 접바둑 UI:** 유저가 선택한 레벨이 1~5일 경우에만 노출되는 '접바둑 개수(0~6)' 선택 버튼 로직 구현 (`AnimatePresence` 애니메이션 포함).
- ✅ **2-4. 상태 전달:** 선택된 설정값(보드 크기, 레벨, 접바둑 개수)을 쿼리 파라미터로 묶어 `/game?size=9&level=3&handicap=2` 형태로 전달.

## Phase 3: 게임 스크린 개편 (캐릭터 배치 및 초기화)

- **목표:** 실제 바둑을 두는 듯한 몰입감과 기본 UI 구성.
- ✅ **3-1. 바둑판 래핑:** `app/game/page.tsx` 레이아웃을 모바일 우선 세로형으로 재구성 및 Sabaki `GoBoard` 컴포넌트 렌더링.
- ✅ **3-2. 캐릭터 이미지 배치:** 바둑판 영역 우측 하단에 `assets/images/level_1.png` 이미지를 `absolute` 속성으로 자연스럽게 겹치도록 배치.
- ✅ **3-3. 하단 컨트롤러 구현:** 화면 하단 3열 그리드에 [무르기], [한수쉼], [기권] 버튼 UI 배치.
- ✅ **3-4. 접바둑 렌더링:** 메뉴에서 전달받은 `handicap` 쿼리 파라미터로 `GoBoard` 초기화 시 표준 좌표에 흑돌 미리 배치. 접바둑 2개 이상이면 백(AI)이 먼저 착수.

## Phase 3-extra: 메인 메뉴(Menu) 2단 UI 개편 및 난이도 축소

- **목표:** 좌측은 게임 모드/설정, 우측은 유저 프로필(대시보드)로 분리하고, 타겟 유저에 맞춰 난이도를 1~5 스케일로 조정.
- ✅ **3-extra-1. 2단 레이아웃 분리:** `app/menu/page.tsx` 화면을 좌측(메뉴 및 상세 설정)과 우측(상태 대시보드)으로 나누는 반응형 `md:grid-cols-2` 레이아웃 구현.
- ✅ **3-extra-2. 우측 프로필 대시보드 렌더링:** `localStorage`의 `bbulmung_progress` 키에서 `currentLevel`, `currentExp`, `requiredExp`를 읽어 레벨 + EXP 프로그레스 바 및 `level_1.png` 캐릭터 이미지(부유 애니메이션) 렌더링.
- ✅ **3-extra-3. 좌측 주요 모드 탭 구현:** "AI와의 대결" / "사활풀이" 두 탭을 배치하고, 선택에 따라 하단 폼이 `AnimatePresence`로 동적 전환.
- ✅ **3-extra-4. 대결 상세 설정 UI (난이도 1~5):** 레벨 1~5 버튼, 접바둑 0~6 버튼(전 레벨 노출), 바둑판 크기(9/13/19) 선택, [시작하기] 버튼.
- ✅ **3-extra-5. 라우팅 연동:** 대결 → `/game?size=&level=&handicap=`, 사활풀이 → `/tsumego` 라우팅 연결.

## Phase 4: 외부 AI 서버 구축 (FastAPI + GNU Go on Cloud Run)

- **목표:** 클라우드 환경에 Stateless(무상태) 바둑 AI 서버 배포.
- **4-1. 프로젝트 세팅:** `backend/` 폴더 생성, Python 가상환경 세팅 및 `FastAPI`, `uvicorn` 설치.
- **4-2. GNU Go 래퍼 클래스:** Python `subprocess`를 이용해 GNU Go 프로세스를 열고, GTP 명령어(레벨 설정, SGF 로드, 다음 수 생성 등)를 주고받는 유틸리티 함수 작성.
- **4-3. API 엔드포인트 구현:** `POST /api/play` 라우터 생성. 클라이언트로부터 `sgf`, `level`, `handicap` 데이터를 받아 GNU Go에 주입하고 결과 좌표를 반환하는 로직 작성.
- **4-4. Dockerizing:** `python:3.11-slim` 기반으로 OS 패키지 `gnugo`를 설치하고 FastAPI 앱을 실행하는 `Dockerfile` 작성.
- **4-5. 클라우드 배포:** `gcloud run deploy` 명령어를 통한 배포 및 CORS 미들웨어 설정. 결과 URL을 프론트엔드 `.env.local`에 `NEXT_PUBLIC_API_URL`로 등록.

## Phase 5: 클라이언트 - 서버 AI 대전 및 무르기 연동

- **목표:** 프론트엔드와 백엔드를 연결하여 실제 대국 사이클 및 롤백 완성.
- **5-1. API 통신 모듈:** 클라이언트에서 서버의 `/api/play` 엔드포인트로 SGF 트리와 메타데이터를 전송하는 비동기 fetch 함수 작성.
- **5-2. 착수 및 로딩 상태 연동:** 유저가 바둑판에 돌을 두면 로딩 스피너/텍스트를 띄우고 API 통신 모듈을 호출하는 로직 연결.
- **5-3. AI 응답 렌더링:** 서버로부터 수신한 좌표 데이터를 Sabaki 보드의 백돌 착수 이벤트로 연결.
- **5-4. 클라이언트 무르기 로직:** [무르기] 버튼 클릭 시, 현재 관리 중인 기보(SGF 트리)에서 마지막 2수(서버 응답 수 + 내 착수)를 삭제하고 화면을 롤백(리렌더링)하는 로직 구현. (서버 통신 불필요)

## Phase 6: AI 기반 사활 퀴즈 모드

- **목표:** 동적 AI 반응이 포함된 사활 문제 풀이 화면.
- **6-1. 사활 페이지 세팅:** `app/tsumego/page.tsx` 생성 및 `public/data/tsumego/` 경로의 SGF 파일을 파싱하여 초기 화면에 렌더링하는 함수 구현.
- **6-2. 정답 검증 로직:** 유저 착수 시 SGF에 사전 정의된 정답 트리(Next Node)와 일치하는지 검사하는 분기 처리.
- **6-3. 동적 오답 대응 (AI):** 정답 트리를 벗어난 오답일 경우, Cloud Run API(`POST /api/play`)를 호출하여 AI가 해당 수에 대해 방어하거나 공격하는 응징수 렌더링.

## Phase 7: 경험치(EXP) 및 레벨업 시스템

- **목표:** 동기 부여를 위한 상태 관리 및 UI.
- **7-1. 상태 관리소 구축:** `localStorage` (또는 Zustand)에 `{ currentLevel: 1, currentExp: 0, requiredExp: 100 }` 형태의 스키마 및 업데이트 액션 정의.
- **7-2. EXP 부여 로직:** 대국 종료(승기/기권) 시 난이도에 따른 계산식 적용 및 사활 정답 시 고정 EXP를 상태 스토어에 추가하는 로직 작성.
- **7-3. 헤더 프로그레스 바 UI:** 글로벌 레이아웃 상단에 현재 레벨과 경험치 진행도(Progress Bar)를 시각적으로 보여주는 컴포넌트 추가.
- **7-4. 레벨업 트리거:** 경험치가 `requiredExp`를 초과할 경우 `currentLevel`을 증가시키고 잔여 경험치를 이월하는 로직 적용.

## Phase 8: 최종 폴리싱 (사운드 및 애니메이션)

- **목표:** 앱의 완성도를 높이는 시청각 피드백 추가.
- **8-1. Web Audio 세팅:** 착수음(`click.mp3` 등) 오디오 파일을 `public`에 추가하고 브라우저 제약을 고려한 오디오 재생 유틸리티 훅(`useSound`) 작성.
- **8-2. 착수음 연동:** Sabaki 컴포넌트의 착수 성공 이벤트와 서버 AI 착수 이벤트 발생 시 오디오 재생 로직 연결.
- **8-3. 레벨업/정답 애니메이션:** `framer-motion`을 사용하여 사활 정답 시 나타나는 팝업 메시지 및 레벨업 시 파티클(폭죽) 애니메이션 컴포넌트 구현.
- **8-4. 반응형 레이아웃 튜닝:** 모바일/태블릿 가로 및 세로 모드에서 버튼 터치 영역 및 바둑판, 캐릭터 이미지 겹침 현상이 없는지 최종 CSS 조정.
