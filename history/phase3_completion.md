# Phase 3 완료 보고서 및 작업 히스토리

## 🎯 완료된 주요 작업 (2026-05-07)

### 1. 치명적인 버그 해결 및 안정성 확보
* **Call Stack 초과 버그 해결**: '한수쉼(패스)' 버튼 클릭 시 `GoBoard`와 `GamePage` 간의 콜백 지옥(Infinite Loop)으로 인해 발생하던 `Maximum call stack size exceeded` 에러를 콜백 분리를 통해 완전히 해결했습니다.
* **상태 업데이트 충돌 방지**: `GoBoard`의 프로그램 착수(`playAt`) 시 발생하던 React의 `Cannot update a component while rendering a different component` 에러를 동기적 상태 업데이트 로직으로 개편하여 안정성을 확보했습니다.

### 2. 혁신적인 사활(Dead stones) 및 계가 시스템 도입
* **GNU Go 엔진 한계 극복**: `gnugo.wasm`의 내장 `_score` API가 브라우저 환경에서 런타임 크래시(`exit 1`)를 일으키는 문제를 확인하고 자체 계가 시스템으로 전환했습니다.
* **Next.js Server Action 도입**: 빌드 충돌과 파일 시스템(`fs`) 접근 제약으로 인해 포기했던 `@sabaki/deadstones`를 **클라이언트가 아닌 서버 백엔드 액션(`app/actions/scoring.ts`)**에서 실행되도록 아키텍처를 변경했습니다.
* **fetch(null) 크래시 우회**: Next.js 빌드 시 로컬 WASM 파일을 찾지 못해 발생하던 코어 버그를 방지하기 위해, CDN(unpkg) 기반의 `deadstones.useFetch()` 폴백(Fallback) 라우팅을 구성하여 에러를 완벽히 차단했습니다.
* **결과**: 외부 설치나 환경 설정 없이, 게임 종료 시 서버에서 100회의 몬테카를로 시뮬레이션을 돌려 죽은 돌을 정확히 솎아내고 중국식 룰(Area Scoring)로 승패를 판정하는 완벽한 계가 시스템이 완성되었습니다.

### 3. 사용자 경험(UX) 및 인터페이스 개선
* **토스트 알림 추가**: AI가 둘 곳이 없어 '한수쉼(패스)'을 했을 때, 플레이어가 이를 직관적으로 인지할 수 있도록 `framer-motion`을 활용한 부드러운 팝업 알림을 추가했습니다.
* **친화적인 용어 변경**: 딱딱한 '컴퓨터'라는 단어를 현대적인 'AI'로 일괄 교체했습니다.
* **동적 난이도 조절 패널 (어린이 맞춤형)**:
  * 게임 중 언제라도 AI의 실력을 3단계로 조절할 수 있는 직관적인 UI(🐥 병아리, 🐶 강아지, 🐯 호랑이)를 신설했습니다.
  * 레벨 변경 시 게임의 맥을 끊지 않고 백그라운드 Web Worker 파라미터(`--level`)만 부드럽게 교체하여 다음 수부터 즉시 난이도가 적용되도록 구현했습니다.
